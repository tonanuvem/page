Referência Rápida Docker e Docker Compose
1. Comandos Básicos do Docker
Informações e Versão
docker version Mostra a versão do Docker instalada.
docker info Exibe informações detalhadas sobre o ambiente Docker.
Gerenciando Contêineres
docker run -it --rm --name <nome_do_container> <nome_da_imagem> <comando> Cria e inicia um novo contêiner a partir de uma imagem.
docker ps -a Lista todos os contêineres, incluindo os que estão parados.
docker start <container_id_ou_nome> Inicia um contêiner parado.
docker stop <container_id_ou_nome> Para um contêiner em execução.
docker restart <container_id_ou_nome> Reinicia um contêiner.
docker rm <container_id_ou_nome> Remove um contêiner parado.
docker rm $(docker ps -a -q) Remove todos os contêineres parados.
Gerenciando Imagens
docker images Lista todas as imagens baixadas localmente.
docker pull <nome_da_imagem>:<tag> Baixa uma imagem do Docker Hub (ou de outro registro).
docker build -t <nome_da_imagem> . Constrói uma imagem a partir de um Dockerfile.
docker rmi <image_id_ou_nome> Remove uma imagem.
docker rmi $(docker images -a -q) Remove todas as imagens.
Logs e Inspeção
docker logs -f <container_id_ou_nome> Exibe os logs de um contêiner em tempo real.
docker inspect <container_id_ou_nome> Exibe informações detalhadas (JSON) sobre um contêiner.
docker exec -it <container_id_ou_nome> bash Executa um comando (ex: bash) dentro de um contêiner em execução.
2. Comandos do Docker Compose
Gerenciando Serviços
docker compose up -d Cria e inicia os serviços definidos no `docker-compose.yml` em segundo plano.
docker compose down Para e remove contêineres, redes e volumes criados pelo `up`.
docker compose build Reconstrói as imagens dos serviços.
docker compose ps Lista os contêineres dos serviços.
docker compose logs -f Exibe os logs de todos os serviços em tempo real.
docker compose run <nome_do_serviço> <comando> Executa um comando único em um novo contêiner.
docker compose stop <nome_do_serviço> Para um serviço específico.
docker compose start <nome_do_serviço> Inicia um serviço parado.
Observações Importantes
  • Substitua os placeholders: Lembre-se de substituir os valores entre <> pelos seus valores reais.
  • docker-compose.yml: Os comandos do Docker Compose devem ser executados no mesmo diretório onde o arquivo está localizado.
  • Versão do Docker: A partir da versão 1.27 do Docker, o comando docker-compose foi integrado ao CLI principal, sendo substituído por docker compose.
  • Limpeza: Para liberar espaço, use docker system prune para remover contêineres, redes, imagens e volumes não utilizados.
  • Documentação: Para informações detalhadas, consulte a documentação oficial do Docker.
